Class Recurrence

Modèle de gestion des récurrences bancaires (modèles générateurs de transactions).

Une récurrence est un modèle qui génère automatiquement des transactions à intervalle régulier. Exemples : loyers, salaires, abonnements, prélèvements mensuels, etc.

Les occurrences générées sont stockées dans la table transactions avec un recurrence_id pointant vers ce modèle.

calculerProchaineExecution()

Recurrence.php : 184

Calcule la prochaine date d'exécution d'une récurrence

public static calculerProchaineExecution(array<string|int, mixed> $recurrence) : string|null

Applique l'intervalle et la fréquence pour calculer la date suivante. Gère la tolérance weekend (décalage au vendredi précédent ou lundi suivant).

Fréquences supportées :

  • quotidien : +N jours
  • hebdomadaire : +N semaines
  • mensuel : +N mois
  • trimestriel : +N*3 mois
  • semestriel : +N*6 mois
  • annuel : +N années
Parameters
$recurrence : array<string|int, mixed>

Données de la récurrence

  • string $recurrence['prochaine_execution'] Date actuelle
  • string $recurrence['frequence'] Fréquence de récurrence
  • int $recurrence['intervalle'] Multiplicateur
  • string $recurrence['tolerance_weekend'] Gestion des weekends
Tags
example

$recurrence = [ 'prochaine_execution' => '2024-01-15', 'frequence' => 'mensuel', 'intervalle' => 1, 'tolerance_weekend' => 'jour_ouvre_suivant' ]; $prochaineDate = Recurrence::calculerProchaineExecution($recurrence); // Résultat: '2024-02-15' (ou '2024-02-16' si 15/02 est un samedi)

Return values
string|null —

Date de prochaine exécution (YYYY-MM-DD) ou null si invalide

executerRecurrence()

Recurrence.php : 263

Exécute une récurrence en créant une nouvelle transaction

public static executerRecurrence(int $recurrenceId) : int|null

Processus complet :

  1. Vérifie que la récurrence existe et est active
  2. Crée une nouvelle transaction dans la table transactions
  3. Lie la transaction à la récurrence via recurrence_id
  4. Met à jour la récurrence (derniere_execution, prochaine_execution, nb_executions)
  5. Désactive automatiquement si limite atteinte ou date_fin dépassée
Parameters
$recurrenceId : int

ID de la récurrence à exécuter

Tags
example

$recurrences = Recurrence::getAExecuter(); foreach ($recurrences as $rec) { $newId = Recurrence::executerRecurrence($rec['id']); if ($newId) { echo "Transaction créée : ID {$newId}\n"; } }

getAExecuter()

Récupère les récurrences dont la prochaine exécution est arrivée

public static getAExecuter([string|null $dateMax = null ]) : array<string|int, mixed>

Filtre les récurrences qui doivent être exécutées :

  • prochaine_execution <= $dateMax
  • Actives (recurrence_active = 1)
  • Pas encore expirées (date_fin non atteinte)
  • Nombre d'exécutions max non atteint

Utilisé par le système d'automatisation des récurrences.
